### 1. 混淆导致方法名不可识别，undefined error 崩溃

- 修改日期：2026-03-07
- 版本：0.72.128
- 问题描述：代码混淆后 TurboModule 导出的方法名被改写，运行时按原始名称查找失败，最终出现 undefined error 并触发崩溃
- 影响模块：JS 混淆 / TurboModule
- 提交 / PR：[2bdea79bb](https://gitcode.com/OpenHarmony-RN/ohos_react_native/commit/2bdea79bba269008519b944dafb66495915902bd) / [!2341](https://gitcode.com/OpenHarmony-RN/ohos_react_native/merge_requests/2341)
- 详细修复内容：修复混淆后方法名不可识别导致的 undefined error 崩溃，保障 TurboModule 在混淆场景下仍可正常调用。

### 3. 自定义组件以 RCT 为前缀命名导致崩溃

- 修改日期：2025-10-26
- 版本：0.72.101
- 问题描述：自定义组件名称使用 RCT 前缀时，组件名映射逻辑与系统组件命名规则发生冲突，注册或查找阶段解析失败导致崩溃
- 影响模块：ComponentRegistry
- 提交 / PR：[50800a854](https://gitcode.com/OpenHarmony-RN/ohos_react_native/commit/50800a8541688fc90f2862fa8b625608e86ddf1e) / [!1734](https://gitcode.com/OpenHarmony-RN/ohos_react_native/merge_requests/1734)
- 详细修复内容：修复自定义组件命名以 RCT 为前缀时的兼容性问题，避免注册或查找阶段崩溃。

### 4. HSP 目录下加载 bundle 崩溃

- 修改日期：2025-04-24
- 版本：0.72.67
- 问题描述：bundle 部署在 HSP 目录结构下时，Bundle Loader 仍按普通 HAR/HAP 路径解析资源位置，导致实际 bundle 文件无法定位，加载失败后直接进入崩溃分支
- 影响模块：Bundle Loader
- 提交 / PR：[de132df4a](https://gitcode.com/OpenHarmony-RN/ohos_react_native/commit/de132df4a94f3761bd025e0d6aa7556e54ff26c7) / [!864](https://gitcode.com/OpenHarmony-RN/ohos_react_native/merge_requests/864)
- 详细修复内容：修复 HSP 目录场景下 bundle 加载路径不兼容导致的崩溃。

# 04-资源泄漏

本文件汇总该版本中归类为资源泄漏的历史修复，重点包括内存泄漏、句柄未释放和长期运行后的资源累积问题。

<a id="c56478622"></a>

### 1. TurboModule 与网络请求回调内存泄漏

- 修改日期：2026-01-08
- 版本：0.82.3
- 问题描述：HTTP 请求结束后回调没有及时清理，`AppearanceTurboModule` 和 `NetworkingTurboModule` 析构时也缺少资源释放，长时间运行会持续累积内存。
- 影响模块：NetworkingTurboModule / AppearanceTurboModule / HTTP callbacks
- 提交 / PR：[c56478622](https://gitcode.com/OpenHarmony-RN/ohos_react_native/commit/c56478622aee23466d20fcc72fee4ea876cf2d64) / [!1990](https://gitcode.com/OpenHarmony-RN/ohos_react_native/merge_requests/1990)
- 详细修复内容：为请求回调补充完成后的清理逻辑，并在相关 TurboModule 析构时释放遗漏资源，消除请求生命周期和模块销毁阶段的泄漏点。
